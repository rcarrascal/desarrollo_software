Características del Sistema

  • @OneToMany: Cliente → Ventas, Proveedor → Compras
  • @ManyToOne: Compra → Proveedor, Venta → Cliente
  • @OneToMany bidireccional: Compra/Venta ↔ Detalles
  • Cascade: Operaciones en cascada para detalles
  • Fetch: Lazy loading para optimización
  • Base de datos H2: Fácilmente migrable a MySQL
